Event Loop: JavaScript can do only one thing at a time. But it can still handle things like waiting for timers or downloading data without stopping everything else. It does this using the event loop.

How it works:

  • JavaScript runs your code line by line (this is called the call stack).
  • When it sees something that takes time (like setTimeout or fetching data), it tells the browser to handle it.
  • When that task is done, its result is put in a queue (waiting line).
  • The event loop checks if JavaScript is done with current work.
  • If yes, it takes the next task from the queue and runs it.
